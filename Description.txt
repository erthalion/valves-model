Функции norm, residual, alloc, del - полностью соответствуют своим названиям
A1, A2 - нелинейная и линейная части оператора A

Функция init_gr (остальные с приставкой _gr вспомогательные) - нужна для автоматического расчета "матрицы влияния оператора". Ситуация следующая: Пусть оператор A действует на вектор, у которого элемент (i,j,k) равен 1, а остальные 0. В этом случае вектор-результат имеет довольно мало ненулевых элементов, и для его расчета только их и нужно искать. Так вот, индексы этих элементов хранятся в arg[i][j][k], а их количество в carg[i][j][k]. Выполняется все это дело в трехмерном случае крайне долго, поэтому приходится выкручиваться (см. init).
Очевидный пример использования смотрите в residual(i,j,k), где пересчитывается вектор невязки после изменения (i,j,k) компоненты вектора неизвестных.

функция my_init_gr - задание матрицы влияния вручную. Она отдельно пишется каждого оператора.

Дальше идут функции итерационного метода неполной аппроксимации.
cubic - решение кубического уравнения
tau_iter - первый шаг
alpha_iter - второй шаг
speed_first, speed_work - ускорение сходимости, выполняемое через заданное число шагов метода (см. run).

Дальше - функции вывода информации.

print_texplot - окончательный вывод с уже сынтерполированными значениями узлов
print_info - вывод сопровождающей информации (типа текущей невязки)

Инициализирующие функции
X_init - задание массивов координат узлов и шагов
G_init - маска узлов
vars_init - параметры рассчета
init - обратите внимание на три участка кода: для запуска автоматической процедуры рассчета матрицы влияния с сохранением всех индексов в файл, для чтения матрицы влияния из уже подготовленного файла, для запуска функции расчета матрицы влияния вручную. Выполняется только один из этих блоков.

run - основной цикл.
